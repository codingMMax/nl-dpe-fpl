# FC & DIMM Verification Infrastructure

## Overview

This directory contains RTL behavioral models, testbenches, and alignment logs
that verify the IMC simulator's latency/energy model against cycle-accurate
RTL simulation. The goal: **every pipeline stage in the simulator has a 1:1
match with the RTL cycle count**.

## Behavioral Models

### `dpe_stub.v` — Behavioral DPE Hard Block

Cycle-accurate model of the DPE (Digital Processing Element) crossbar:

| Parameter | Description | Values |
|-----------|-------------|--------|
| `KERNEL_WIDTH` | Input elements per VMM pass | Workload-dependent |
| `NUM_COLS` | Crossbar columns (output elements) | 128 (Proposed) |
| `DPE_BUF_WIDTH` | Bits per w_buf_en strobe | 16 or 40 |
| `COMPUTE_CYCLES` | Bit-serial pipeline delay | 44 (ADC), 3 (ACAM) |
| `ACAM_MODE` | Nonlinear function | 0=none, 1=exp, 2=log |

**Packed transfers**: Each `w_buf_en` strobe carries `DPE_BUF_WIDTH/8` int8
elements. `LOAD_STROBES = ceil(KERNEL_WIDTH / elems_per_strobe)`.
`OUTPUT_CYCLES = ceil(NUM_COLS / elems_per_strobe)`.

**Pipeline stages** (matches simulator's `gemm_pipeline_profile`):
```
S_IDLE → S_LOAD (LOAD_STROBES cycles)
       → S_WAIT_EXEC (1 cycle: reg_full → nl_dpe_control)
       → S_COMPUTE (COMPUTE_CYCLES cycles)
       → S_OUTPUT (OUTPUT_CYCLES cycles, dpe_done=1)
       → S_DRAIN (1 cycle) → S_IDLE
```

**ACAM**: Applied after VMM in S_WAIT_EXEC. exp(x) ≈ 1 + x + x²/2.
Weight memory zero-initialized (identity weights loaded by testbench).

### Supporting Modules (from `gemv_1_channel.v`)

- `sram` — Parameterized registered-read SRAM
- `conv_controller` — FC layer controller (SRAM interleave, w_buf_en generation)
- `controller_scalable` — V>1 multi-DPE controller
- `conv_layer_single_dpe` — FC DPE wrapper (SRAM + controller + DPE)
- `global_controller` — Top-level stall/valid handshake

## FC Layer Verification

### RTL Under Test

Generated by `nl_dpe/gen_gemv_wrappers.py`:
- `block_comp_apr_11/rtl/setup{0-5}/fc_{K}_{N}_{R}x{C}_{conv}_dw{bw}.v`
- Module hierarchy: `fc_top → conv_layer_single_dpe → dpe`
- Packed SRAM: `DATA_WIDTH = dpe_data_width`, `DEPTH = ceil(K/epw) + 1`
- Controller `KERNEL_WIDTH = ceil(K/epw)` (packed word count)

### Testbenches

| File | Type | What it verifies |
|------|------|-----------------|
| `tb_dpe_vmm.v` | Functional | VMM correctness: W×x = [50, 60] for known inputs |
| `tb_dimm_functional.v` | Functional | Standalone ACAM exp: identity×[1,2,3,4] → [2,5,8,13] |
| `tb_alignment.v` | Latency | V=1 FC: pre-load SRAM, measure read/compute/output cycles |
| `tb_alignment_v4h2.v` | Latency | V>1 FC: pre-load per-row SRAMs, measure parallel DPE pipeline |
| `run_alignment.sh` | Runner | Runs all 6 setups × 2 workloads, dumps alignment_log.txt |

### Alignment Method (FC)

1. **Pre-load SRAM** via hierarchical memory write (bypasses input feed stage)
2. **Set controller `next_address`** to indicate data available (`memory_flag=1`)
3. **Measure timestamps** at internal signal transitions:
   - `T_wbuf_first`: first `w_buf_en` (SRAM→DPE read starts)
   - `T_regfull`: `reg_full` asserted (DPE buffer full)
   - `T_done_first`: first `dpe_done` (output serialize starts)
   - `T_done_last`: last `dpe_done` (output serialize ends)
4. **Compute per-stage cycles**:
   - read = T_regfull − T_wbuf_first
   - compute = T_done_first − T_regfull
   - output = T_done_last − T_done_first + 1

## DIMM Pipeline Verification

### RTL Under Test

Generated by `gen_dimm_wrapper.py` using `gen_attention_wrapper.py`:
- `fc_verification/rtl/dimm_pipeline_d64_c128.v`
- Module: `dimm_score_matrix` with direct DPE instantiation
- Architecture: CLB byte-wise add → DPE(I|exp) → masked accumulator → score SRAM

### Configuration

| Parameter | IMC Simulator | RTL | Match |
|-----------|--------------|-----|-------|
| Crossbar R | 1024 | 1024 (defparam) | ✓ |
| Crossbar C | 128 | 128 (NUM_COLS) | ✓ |
| dpe_buf_width | 40 | 40 (DPE_BUF_WIDTH) | ✓ |
| d_head | 64 | 64 (d parameter) | ✓ |
| K_id | 2 (C//d) | 2 (dual-identity FSM) | ✓ |
| packed_width | 128 (K_id×d) | 128 (KERNEL_WIDTH) | ✓ |
| COMPUTE_CYCLES | 3 (ACAM) | 3 | ✓ |
| LOAD_STROBES | 26 (ceil(128/5)) | 26 | ✓ |
| OUTPUT_CYCLES | 26 (ceil(128/5)) | 26 | ✓ |
| DPE count | 1 (h_dimm=1) | 1 | ✓ |
| Scores per pass | 2 (K_id) | 2 (S_WRITE_B) | ✓ |

### Testbenches

| File | Type | What it verifies |
|------|------|-----------------|
| `tb_dimm_pipeline_functional.v` | Functional | Score matrix correctness with K_id=2, d=64, C=128 |
| `tb_dimm_pipeline_latency.v` | Latency | Per-DPE-pass cycle count, w_buf_en/dpe_done pulse counts |

### Alignment Method (DIMM)

1. **Pre-load identity weights** into DPE via hierarchical access (128×128 diagonal)
2. **Pre-load Q SRAM** (one-hot query, 13 packed words)
3. **Pre-load K SRAMs** (identity keys, padded per-key at word-aligned offsets)
4. **Set FSM to S_COMPUTE** directly (bypass Q/K loading phase)
5. **Measure score write timestamps** and DPE signal counts

### DIMM Results: IMC Simulator vs RTL

**Test: N=4, d_head=64, C=128, K_id=2 (dual-identity)**

#### Functional Correctness

| Score | Expected | RTL | Match |
|-------|----------|-----|-------|
| S[0] | 68 = exp(2) + 63×exp(0) | 68 | ✓ |
| S[1] | 66 = 2×exp(1) + 62×exp(0) | 66 | ✓ |
| S[2] | 66 | 66 | ✓ |
| S[3] | 66 | 66 | ✓ |

#### Latency Alignment (per DPE pass = per score pair)

| Stage | IMC Simulator (cycles) | RTL (cycles) | Match |
|-------|----------------------|--------------|-------|
| SRAM→DPE read | 26 (ceil(128×8/40)) | 26 (w_buf_en pulses) | ✓ |
| DPE compute | 3 (ACAM bit-serial) | 3 + 4 overhead | +4 |
| DPE output serialize | 26 (ceil(128×8/40)) | 26 (dpe_done pulses) | ✓ |
| CLB add | 1 | (overlapped with feed) | — |
| CLB reduce | 6 (ceil(log2(64))) | (overlapped with accumulate) | — |
| **Per DPE pass** | **62** (1+55+6) | **65** | **+3** |
| **Per score pair** | **62** | **65** | **+3** |

The +3 cycles/pass overhead is FSM coordination:
- 2 cycles: SRAM read latency (address register + registered read)
- 1 cycle: drain wait + score write

### Data Path Comparison

| Component | IMC Simulator Model | RTL Implementation |
|-----------|--------------------|--------------------|
| Input packing | `_dpe_buf_fill_row(C)`: ceil(C×8/dpe_bw) accesses | Packed SRAM words (epw=5 int8/word), `w_buf_en` strobes |
| CLB addition | Implicit (1 cycle in cycles_per_row) | Byte-wise parallel adder (genvar, no carry between lanes) |
| DPE identity | `dimm_nonlinear()`: VMM with identity weights | `dpe_stub.v`: weight[r][c] = δ(r,c), VMM + ACAM |
| K_id packing | `K_id = C//K`, amortizes DPE pass across K_id elements | Dual-identity FSM: feed vec A then B, separate masked accumulators |
| Output serialize | `ceil(C×8/dpe_bw)` cycles | `OUTPUT_CYCLES` packed words from DPE |
| Accumulation | Implicit in energy model (per-element) | Masked byte-wise sum: A (cols 0..d-1), B (cols d..2d-1) |
| Score write | Not modeled (instantaneous) | FSM S_WRITE_SCORE + S_WRITE_B (2 cycles for dual) |
| Memory model | BRAM read/write via `memory.latency()` | Registered SRAM with 2-cycle read latency |

### Assumptions

| Assumption | Simulator | RTL | Notes |
|-----------|-----------|-----|-------|
| All C columns fire per pass | Yes (energy = full crossbar) | Yes (identity spans full C) | Full utilization |
| SRAM read latency | Not modeled per-stage (bulk) | 2 cycles (addr reg + SRAM reg) | RTL has pipeline overhead |
| DPE drain between passes | Not modeled | 1 cycle (S_DRAIN) | RTL waits for MSB_SA_Ready |
| Packed I/O | ceil(bytes×8/dpe_bw) accesses | Same formula via LOAD_STROBES/OUTPUT_CYCLES | Exact match |
| Controller overhead | Not modeled | +4 cycles (reg_full→nl_dpe_control) | Constant, independent of config |
